# Make a Plan for Tasks Skill

## When to Use

- User says "Make a Plan for tasks"
- User asks to analyze pending tasks
- User requests strategic planning or prioritization
- User wants to understand task execution order

## Procedure

1. **Open the Needs_Action folder** and list all `.md` task files
2. **Read each task file** including frontmatter and content
3. **Analyze each task for:**
   - Task type (from `type` field in frontmatter)
   - Priority level (from `priority` field: high/medium/low)
   - Related files (from `related_files` field)
   - Content complexity (from task description and checklist)
4. **Create a new plan file** in `Plans/` folder:
   - Filename format: `Plan_<timestamp>.md` (e.g., `Plan_2026-02-24_15-30-00.md`)
   - Use the structured format below
5. **Do NOT execute tasks** - This is planning only

## Planning Guidelines

### Prioritization Rules

1. **High priority tasks** ‚Üí Process first
2. **File review tasks** ‚Üí Often provide context for other tasks, process early
3. **Similar task types** ‚Üí Batch together for efficiency
4. **Tasks with dependencies** ‚Üí Order by dependency chain

### Risk Identification

Look for:
- Ambiguous task descriptions
- Missing information in frontmatter
- Tasks requiring external access or permissions
- Potentially destructive actions (deletions, overwrites)
- Large or complex files

### Strategy Recommendations

Consider:
- **Batching:** Group similar tasks (all file reviews, all documents, etc.)
- **Dependencies:** Some tasks may provide context for others
- **Time estimates:** Rough estimate for completion
- **Resources needed:** Any special tools or access required

## Important Rules

- **This is planning ONLY** - Do not execute or modify tasks
- **Do not move files** - Tasks stay in Needs_Action
- **Be concise but thorough** - One page maximum for plan
- **Highlight priorities clearly** - Use visual indicators (üî¥ high, üü° medium, üü¢ low)
- **Flag unclear items** - If a task is ambiguous, note it in Risks section

## Troubleshooting

- **If Needs_Action is empty:** Report "No pending tasks to plan"
- **If task files are corrupted:** Note in Risks section, skip analysis
- **If Plans folder doesn't exist:** Create it automatically
- **If timestamp format fails:** Use simple format `Plan_<unix_timestamp>.md`
