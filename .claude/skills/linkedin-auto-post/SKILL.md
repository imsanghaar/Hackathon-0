# LinkedIn Auto-Post Skill

## When to Use

- User creates a Plan.md that includes a LinkedIn post task
- A `linkedin_post_request.md` file appears in `Needs_Action/` folder
- Automated business posting for sales generation
- Scheduled social media updates

## Procedure

1. **Check for LinkedIn Post Requests**
   - Scan `AI_Employee_Vault/Needs_Action/` for files containing "linkedin" or "post"
   - Look for frontmatter with `type: linkedin_post`

2. **Extract Post Content**
   - Read the post text from the file
   - Extract hashtags, mentions, and media references

3. **Post to LinkedIn via MCP**
   - Navigate to LinkedIn
   - Click "Start a post"
   - Enter post content
   - Click "Post" button

4. **Log and Archive**
   - Log the successful post
   - Move request file to `Done/` folder
   - Save post URL for reference

## Important Rules

- **Human Approval:** For sensitive posts, require approval first
- **Rate Limiting:** Max 3 posts per hour to avoid LinkedIn restrictions
- **Content Validation:** Check for prohibited content before posting
- **Error Recovery:** Retry failed posts up to 3 times
- **Logging:** Always log post attempts and results

## File Paths

- **Input:** `AI_Employee_Vault/Needs_Action/linkedin_post_*.md`
- **Output:** `AI_Employee_Vault/Done/linkedin_post_*.md`
- **Log:** `Logs/linkedin_posts.log`
- **Script:** `.claude/skills/linkedin-auto-post/scripts/auto_post.py`

## Integration with Task Planner

When Task Planner creates a plan that includes LinkedIn posting:

1. Creates `Plan_<filename>.md` in Needs_Action/
2. If plan includes "LinkedIn post" step, also creates `linkedin_post_request.md`
3. Scheduler detects and processes the request
4. Posts automatically or waits for approval based on sensitivity

## Running the Auto-Poster

**Manual execution:**
```bash
python .claude/skills/linkedin-auto-post/scripts/auto_post.py
```

**Dry run (preview):**
```bash
python .claude/skills/linkedin-auto-post/scripts/auto_post.py --dry-run
```

**Continuous monitoring:**
```bash
python .claude/skills/linkedin-auto-post/scripts/auto_post.py --watch --interval 300
```

## Example Workflow

1. **User drops file in Inbox:**
   ```
   Inbox: "Q1_Sales_Campaign.md"
   ```

2. **Task Planner creates plan:**
   ```markdown
   ## Step 3: Create LinkedIn Post
   - Action: Post about Q1 sales campaign
   - Details: Use linkedin-auto-post skill
   - Expected: Post published to LinkedIn
   ```

3. **LinkedIn Auto-Post triggers:**
   - Detects LinkedIn post task in plan
   - Creates `linkedin_post_request.md`
   - Executes post via Playwright MCP
   - Logs result

4. **Files moved to Done:**
   ```
   Done: Plan_Q1_Sales_Campaign.md
   Done: linkedin_post_request.md
   ```

## Troubleshooting

| Issue | Solution |
|-------|----------|
| Not logged into LinkedIn | Manually login first, then run auto-poster |
| Post button not found | Refresh page, check for UI changes |
| Rate limit exceeded | Wait 1 hour before next post |
| MCP server not running | Start with `npx @playwright/mcp@latest --port 8808` |

## Security Notes

- Never store LinkedIn credentials in code
- Use environment variables for sensitive data
- Log out after posting if on shared machine
- Review posts before publishing for sensitive content
